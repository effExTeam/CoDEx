ElectroweakRealTripletScalar

This notebook contains tutorial for computation of Wilson Coefficients for Electroweak Real Triplet Scalar model.

Special Functions:

hfvecstrOut
lOutputflow

This model has a real ( hypercharge, Y =0) isospin triplet scalar field. For initialization of the model in hand, we have to provide this information in a file named "UserDefined.m". For that we load the package first following way:

Open a Mathematica notebook file and "save" it in any directory you wish. Saving the notebook file is crucial. Then next step is to load the package.

Loading the package:

CoDEx-0.0

Supratim Das Bakshi, Joydeep Chakrabortty, Sunando K. Patra
IIT Kanpur, India

If the package is not installed in your machine, use the following (url installation - Importing the package from github repository) :
Import["https://github.com/sunandopatra/CoDEx-0.0/raw/master/install.m"] InstallCoDEx[]

If you use this command, you don't need to do Needs["CoDEx`"] for loading. But from next kernel session, we may use Needs["CoDEx`"] for loading the package. Next job is to provide the model specification in "UserDefined.nb" file. Details provided below.

Inputs for loading model:

After loading the package (using needs or direct url installation), this mini-window will appear on the screen:

Clicking on :

the first option - Creates a "UserDefined.nb" file in the NotebookDirectory[] (directory where your created notebook is saved) , and you are expected to edit it according to the procedure mentioned in "UserDefined" guide. Search Mathematica documentation center for "UserDefined" guide.

the second option - Expects that you already have a prepared "UserDefined.m" file saved in some directory and the package will import the "UserDefined.m" file from the directory that you will indicate.

the third option - Expects that you already have a prepared "UserDefined.m" file saved in the NotebookDirectory[] (directory where your created notebook is saved) directory and the package will import the "UserDefined.m" file from NotebookDirectory[].

Now, we write the Lagrangian.

Subsequently, another mini-window will appear on the screen:

Clicking on :

the first option - Expects that you will write the Lagrangian of your model with the use of definition of hfvecs. The package definition of heavy field will be shown in the name of variable, hfvecs. Like this :

Your heavy fields are now defined. They are listed under the name 'hfvecs'. They are:
{{{hf[1, 1], hf[1, 2], hf[1, 3]}}}

Heavy field 'hf'  Color Singlet. Please remember to provide the Isospin Symmetry Generator Matrix for loop Wilson Coefficients.

And using this definition of hfvecs, the Lagrangian should be constructed.

the second option - Expects that you already have a prepared the model Lagrangian in some .m file saved in some directory and the package will import the Lagrangian from the indicated directory.

the third option - Prints a demo Lagrangian. For this choice, you are expected to build the Lagrangian in the notebook file. See "BuildLagrangian" for details.
